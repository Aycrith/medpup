# Soft-Pilot Launch Checklist

## Required Before Public Soft Launch

### Scope and Legal Safety

- [ ] Mission statement finalized.
- [ ] Service boundary policy finalized.
- [ ] Public disclaimer written.
- [ ] Emergency disclaimer written.
- [ ] Prohibited services listed.
- [ ] Donation button disabled unless compliance is cleared.
- [ ] No medical-service language appears on public materials.
- [ ] No clinic/mobile-vet/urgent-care language appears on public materials.

### Pilot Zone

- [ ] Pilot county/ZIPs selected.
- [ ] Animal control jurisdiction identified.
- [ ] Public shelter routing identified.
- [ ] Emergency clinics identified.
- [ ] Out-of-area message written.

### Intake and Case Management

- [ ] Intake form built.
- [ ] Emergency red-flag questions appear early.
- [ ] Consent checkboxes required.
- [ ] Case board created.
- [ ] Statuses configured.
- [ ] Follow-up fields configured.
- [ ] Emergency cases visibly flagged.
- [ ] Test submissions work.

### Resource Directory

- [ ] At least 20 resource entries added.
- [ ] At least 3 emergency clinic options added.
- [ ] At least 1 animal control/shelter contact per jurisdiction added.
- [ ] Last verified date exists for every active resource.
- [ ] Unknown information is marked unknown, not guessed.

### Partner Outreach

- [ ] Outreach tracker created.
- [ ] First 20 targets identified.
- [ ] Vet outreach script ready.
- [ ] Rescue/shelter outreach script ready.
- [ ] Follow-up dates configured.

### Transport

- [ ] Transport protocol drafted.
- [ ] Transport consent draft created.
- [ ] Field safety checklist complete.
- [ ] Basic transport kit priced.
- [ ] Decline criteria written.
- [ ] Incident report process written.

### Dry Run

- [ ] 10 fake test cases completed.
- [ ] Emergency test cases route correctly.
- [ ] Case summaries generate correctly.
- [ ] Follow-up process works.
- [ ] At least 5 workflow fixes documented.

### Pilot Limits

- [ ] Max open cases defined.
- [ ] Max weekly cases defined.
- [ ] No animal housing in month 1.
- [ ] No unsupervised volunteers in month 1.
- [ ] No public fundraising unless compliance is done.
- [ ] No medical services.

## Launch Decision

Launch only if all critical items are checked.

If any of these are missing, do not launch publicly:

- emergency red-flag routing,
- service boundary policy,
- intake consent,
- case board,
- emergency clinic list,
- animal control/shelter routing,
- public disclaimer.

## First-Week Pilot Cap

Recommended:

- 3–5 real cases maximum.
- Manual review of every case.
- Same-day review of emergency flags.
- Daily end-of-day case board review.

## First Review

After first 3–5 cases, review:

- Were boundaries clear?
- Did anyone mistake the project for a clinic?
- Were emergency cases delayed?
- Were resources accurate?
- Did partners respond?
- Was follow-up manageable?
- Did any case create unexpected risk?
