# Florida Animal Aid Access Project — Phase 0 Operating Folder

This folder contains the first executable document set for launching a legally conservative animal aid coordination pilot in Florida.

The Phase 0 objective is to create a small, disciplined operating shell before public launch:
- clear mission,
- strict non-veterinary boundaries,
- emergency escalation,
- structured intake,
- case board schema,
- resource directory schema,
- transport protocol,
- partner outreach tracker,
- field safety rules,
- launch checklist.

## Recommended first pilot posture

Launch as a non-veterinary coordination, referral, documentation, and transport-support project.

Do not launch as:
- a veterinary clinic,
- a mobile vet,
- an urgent-care service,
- a rescue intake facility,
- a shelter,
- a telehealth provider,
- a foster network,
- or a public donation-funded charity until compliance is handled.

## Suggested folder use

1. Review `00_PROJECT_INDEX.md`.
2. Finalize `03_PILOT_ZONE.md`.
3. Build the intake form from `06_INTAKE_FORM_SPEC.md`.
4. Build the case board from `07_CASE_BOARD_SCHEMA.md`.
5. Populate `08_RESOURCE_DIRECTORY_SCHEMA.md`.
6. Run dry test cases before accepting real cases.
7. Do not solicit donations until charitable solicitation compliance is resolved.

## Immediate stack suggestion

For the first pilot:
- Landing page: Carrd, Astro, or simple static page.
- Intake form: Tally, Jotform, Google Forms, or custom React.
- Case board: Airtable or Notion.
- Resource directory: Airtable or Google Sheet.
- Phone/SMS: Google Voice first.
- Email: Gmail or Google Workspace.
- Media storage: Google Drive.
- Automation later: Make, Zapier, n8n, or custom scripts.

## Legal note

This folder is an operational planning artifact, not legal advice. Before fundraising, taking custody of animals, using volunteers at scale, running medical events, or creating a veterinary-facing program, consult a qualified Florida attorney and appropriate licensed veterinary professionals.
